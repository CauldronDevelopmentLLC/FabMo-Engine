Function works like "once()" for a state change callbacks is an associative array mapping states to callbacks If the next state change matches a state in the associative array, the callback it maps to is called. If null is specified in the array, this callback is used for any state that is unspecified eg: this.expectStateChange { STAT_END : end_callback, STAT_PAUSE : pause_callback, null : other_callback}; In the above example, when the next change of state happens, the appropriate callback is called in the case that the new state is either STAT_END or STAT_PAUSE. If the new state is neither, other_callback is called.
